This document summarizes the current Apache/PHP/MariaDB wargame site, database credentials, challenge files, and solve-guide deliverables.

- Main page: `/index.html`
- Challenge pages: `/challenges/*.php`
- API path: `/api/*.php`
- Admin page: `/admin/` or `/admin/index.php`
- Current VM URL: `http://100.83.178.9/`
- Apache web root on VM: `/var/www/html`
- Remote working copy on VM: `/home/test/wargame-lab`

## Database Credentials

The PHP connection is defined in `api/db.php`.

```text
DB host: localhost
DB name: wargame_lab
DB user: wargame_app
DB password: <DB_PASSWORD>
DB charset: utf8mb4
```

The schema creates the app user and grants app-level access:

```sql
CREATE USER IF NOT EXISTS 'wargame_app'@'localhost' IDENTIFIED BY '<DB_PASSWORD>';
GRANT SELECT, INSERT, UPDATE, DELETE ON wargame_lab.* TO 'wargame_app'@'localhost';
```

Apply the full schema:

```bash
sudo mysql < /home/test/wargame-lab/db/schema.sql
```

Apply only the four challenge records, flags, and challenge-specific tables:

```bash
sudo mysql < /home/test/wargame-lab/db/challenge_seed.sql
```

## Challenge-Specific Tables And Files

- `blind_notices`: public notice rows for the Blind Notice boolean SQL injection challenge.
- `admin_memos`: stored memo rows and simulated admin review results for the Admin Memo Chain challenge.
- `/assets/vault/flag.txt`: generated by `image-vault.php` from the DB flag if it is missing.
- `/assets/vault/allowed.png/`: directory used by the Image Vault path-filter bypass.

## Admin Page

```text
URL: /admin/ or /admin/index.php
Admin password: set WARGAME_ADMIN_PASSWORD in the server environment.

```

Admin capabilities:

- Create, edit, activate/deactivate, and delete challenge metadata.
- Create and edit flags.
- Generate PHP challenge templates under `/challenges/generated/{challenge_id}.php`.
- Edit generated PHP template content from the admin page.

## Deployment Notes

Useful service commands:

```bash
sudo systemctl status apache2
sudo systemctl status mariadb
sudo systemctl restart apache2
sudo systemctl restart mariadb
```

After copying changed files to `/home/test/wargame-lab`, deploy only public site files to Apache. Do not copy `docs/` into `/var/www/html` because it contains the private solve guide.

```bash
sudo cp /home/test/wargame-lab/.htaccess /var/www/html/
sudo cp /home/test/wargame-lab/index.html /var/www/html/
sudo cp /home/test/wargame-lab/styles.css /var/www/html/
sudo cp /home/test/wargame-lab/app.js /var/www/html/
sudo cp -r /home/test/wargame-lab/api /var/www/html/
sudo cp -r /home/test/wargame-lab/admin /var/www/html/
sudo cp -r /home/test/wargame-lab/challenges /var/www/html/
sudo cp -r /home/test/wargame-lab/assets /var/www/html/
sudo systemctl restart apache2
```

If `docs/` is accidentally copied to Apache, remove it immediately:

```bash
sudo rm -rf /var/www/html/docs
```

For the current challenge set, also apply:

```bash
sudo mysql < /home/test/wargame-lab/db/challenge_seed.sql
```
## Directory Exposure Controls

Apache `.htaccess` intentionally returns 404 for private or directory-only paths such as `/db/`, `/docs/`, `/.git/`, `/.agents/`, `/assets/vault/`, `/api/`, `/assets/`, and `/challenges/generated/`.

Use `/admin/` or `/admin/index.php` for the admin page. Do not publish the solve guide under Apache web root.
## UI And Progress Reset

- The main page shows a tutorial modal on first visit.
- The modal is hidden after the user closes it because `localStorage.tutorialSeen` is set to `1`.
- Users can clear their current team's solved/submission records with the main-page reset button.
- Reset endpoint: `POST /api/reset.php` with `team=<team_name>`.
